Gateway — Single security boundary

JWT is validated once at the gateway using jose. Downstream services receive X-User-Id, X-User-Email, X-User-Name headers and trust them without re-validating. Safe because auth/task services are ClusterIP — zero internet exposure.

Data ownership — no shared DB

task-auth and task-tasks are separate MongoDB databases. The task-service stores userId as a plain String (denormalised), not an ObjectId reference. No cross-service joins — each service owns its data independently.

BullMQ async notifications

When a task status changes, a STATUS_CHANGED job is enqueued in Redis via BullMQ. The HTTP response returns immediately. In production, the worker runs as a separate Deployment with its own HPA. For this demo it co-locates with task-service.

Kubernetes autoscaling

The task-service HPA scales from 2 to 10 pods when CPU > 70% or memory > 80%. MongoDB uses a StatefulSet for stable network identity and a PVC (PersistentVolumeClaim) for durable storage across pod restarts.
